# Release Check

## Zweck

Vor einer Veröffentlichung Risiken, Qualität und Freigaben prüfen.

## Wann verwenden

Vor Deployments, öffentlichen Veröffentlichungen oder größeren Änderungen.

## Inputs

- Release-Umfang
- betroffene Systeme
- Testergebnisse
- offene Risiken

## Ablauf

1. Scope prüfen.
2. QA-Ergebnisse sichten.
3. Security- und Datenschutzrisiken prüfen.
4. offene TODOs bewerten.
5. Freigabe durch Philipp einholen, falls erforderlich.

## Output

- Go / Go mit Einschränkungen / No-Go
- offene Risiken
- Freigabehinweise

## Empfohlene Skills

- qa-engineer
- security-reviewer
