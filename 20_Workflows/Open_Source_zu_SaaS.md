# Open Source zu SaaS

## Zweck

Open-Source-Projekte analysieren und daraus eine differenzierte SaaS-Chance ableiten.

## Wann verwenden

Wenn ein oder zwei GitHub-Projekte als Ausgangspunkt für ein neues SaaS-Produkt dienen könnten.

## Inputs

- ein oder zwei GitHub-Repositories
- gewünschte Zielgruppe
- mögliche Monetarisierung

## Ablauf

1. Repos auswählen.
2. Lizenz prüfen.
3. Optional DeepWiki oder ein ähnliches Tool zur ersten Repository-Analyse nutzen.
4. Lokal mit Coding Agents analysieren.
5. Feature-Matrix erstellen.
6. Kernfeatures, Stärken und Schwächen extrahieren.
7. Feature-Lücken finden.
8. SaaS-Differenzierung definieren.
9. MVP ableiten.
10. Risiken prüfen.

## Output

- Repo-Vergleich
- gute Ideen
- Schwächen oder Lücken
- SaaS-Chance
- Zielgruppe
- MVP
- Risiken
