# Feature Entwicklung

## Zweck

Features kontrolliert von Idee bis Review umsetzen.

## Wann verwenden

Wenn ein bestehendes Projekt eine neue Funktion erhalten soll.

## Inputs

- Feature-Ziel
- Akzeptanzkriterien
- betroffene Nutzerrollen
- technische Einschränkungen

## Ablauf

1. Ziel und Scope klären.
2. Akzeptanzkriterien schreiben.
3. Risiken prüfen.
4. Implementierung planen.
5. Tests und QA durchführen.
6. Ergebnis dokumentieren.

## Output

- Feature-Plan
- Testfälle
- Review-Ergebnis

## Empfohlene Skills

- produktentwicklung
- qa-engineer
- security-reviewer
