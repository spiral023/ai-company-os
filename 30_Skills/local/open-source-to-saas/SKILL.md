# Open Source zu SaaS

## Ziel

Analysiere Open-Source-Projekte und leite daraus ein kommerziell nutzbares SaaS-Konzept ab.

## Standardablauf

1. Repository-Zweck verstehen.
2. Kernfunktionen extrahieren.
3. Stärken und Schwächen analysieren.
4. Zielgruppen und mögliche Käufer identifizieren.
5. SaaS-Erweiterungen vorschlagen:
   - Multi-Tenant
   - Auth
   - Billing
   - Admin Dashboard
   - Integrationen
   - Reporting
   - Hosting
   - Team-Funktionen
6. Differenzierung gegenüber dem Open-Source-Projekt formulieren.
7. MVP und spätere Features definieren.

## DeepWiki-Nutzung

Wenn DeepWiki oder ein ähnliches Tool verfügbar ist, nutze es zur ersten Repository-Analyse. Ergänze danach lokale Analyse mit Coding Agents.

## Ausgabeformat

- Analysierte Repos
- Was machen die Projekte?
- Gute Ideen
- Schwächen oder Lücken
- SaaS-Chance
- Zielgruppe
- MVP
- Zusatzfeatures
- Risiken
- Nächster Schritt
