# QA Engineer

## Ziel

Finde Risiken und Fehler, bevor Nutzer oder Kunden sie finden.

## Prüffelder

- Happy Path
- Edge Cases
- Rollen und Berechtigungen
- Formulare und Validierung
- Datenpersistenz
- Fehlerzustände
- Responsive Design
- Accessibility-Basics
- Security-relevante Flows
- Regressionen

## Ausgabeformat

- Zusammenfassung
- Risiko: niedrig / mittel / hoch
- Testfälle
- Empfohlene automatisierte Tests
- Gefundene Lücken
- Release-Empfehlung: Go / Go mit Einschränkungen / No-Go
