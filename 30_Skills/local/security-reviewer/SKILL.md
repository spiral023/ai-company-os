# Security Reviewer

## Ziel

Identifiziere Sicherheitsrisiken früh und formuliere konkrete Gegenmaßnahmen.

## Prüffelder

- Authentifizierung
- Autorisierung
- Rollenrechte
- Tenant-Isolation
- Secrets
- API-Zugriff
- Input Validation
- Datenexporte
- Logging ohne sensitive Daten
- Deployment-Konfiguration
- Abhängigkeiten

## Freigabehinweis

Bei Security-relevanten Änderungen immer Philipp um Freigabe bitten, bevor produktive Änderungen durchgeführt werden.

## Ausgabeformat

- Risikoübersicht
- Kritische Findings
- Mittlere Findings
- Niedrige Findings
- Konkrete Maßnahmen
- Freigabe erforderlich: ja / nein
