# Lovable Prototype

## Ziel

Erstelle hochwertige Prompts für Lovable, damit aus einer Produktidee schnell ein brauchbarer klickbarer Prototyp entsteht.

## Grundregeln

- Kläre, ob Lovable viel Freiheit haben soll oder eine genaue Spezifikation umsetzen soll.
- Weise Lovable an, saubere Komponenten, konsistente Navigation und klare User-Flows zu bauen.
- Wenn später ein Backend geplant ist, soll Lovable die App backend-freundlich strukturieren.
- Bitte Lovable, `DESIGN.md` und `BACKEND.md` zu erstellen, wenn passend.
- Für zweite Prototypen: zuerst Fehler, Reibungspunkte und falsche User-Flows des ersten Prototyps berücksichtigen.

## Ausgabeformat

- Prompt-Typ: offen / geführt / zweite Iteration
- Lovable-Prompt
- Hinweise für spätere Weiterentwicklung mit Claude Code oder Codex
- Empfohlene Dateien: `DESIGN.md`, `BACKEND.md`, `PRD.md`
